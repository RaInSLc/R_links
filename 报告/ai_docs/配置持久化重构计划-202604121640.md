# 配置持久化重构与路径精准化逻辑计划 (修复保存失效)

针对“保存不生效”的问题，我们经过深入分析发现当前的 Win32 方案在多页面切换和路径处理上存在不稳定性。本计划将通过底层架构重构彻底解决该问题。

## 方案设计

### 1. 内存与路径安全重构
- **修正 C++ 类型错误**：将配置文件路径改为动态成员变量，移除危险的强制转换 (`const_cast`)，确保内存操作符合现代 C++ 标准。
- **锁定绝对路径**：利用 `GetModuleFileNameW` 强制将 `config.ini` 定位在 EXE 文件的同级目录，消除“不知存到哪里去”的歧义。

### 2. 重构保存引擎 (弃用 Win32 API)
- **改用 C++ 文件流**：弃用老旧且自带缓存的 `WritePrivateProfileStringW`，改用全受控的 `std::wofstream` 写入。
- **显式强制落盘**：在写入后调用 `flush()`，确保您的编辑器（如 VS Code）能瞬间捕捉到文件更新。

### 3. 用户反馈强化
- **显式路径提示**：保存成功后，状态栏将直接显示配置文件的**完整绝对路径**，您可以直接按此路径核对。

## 验证计划
- **实时同步校验**：在 IDE 打开 `config.ini` 的情况下点击保存，确认内容立即刷新。
- **跨环境验证**：确保重启程序后，所有改动都能被正确回读并对准界面。
