# Win32 API 字体渲染与 DPI 高清化适配技术指南

本手册记录了在 R 程序安装链接生成器（C++/Win32 版）开发过程中遇到的字体模糊、锯齿及缩放问题的完整解决方案。

---

## 1. 字体锯齿问题 (Jagged Edges)

### 现象描述
文字边缘呈现阶梯状，笔画不够圆润，视觉感极其生硬。

### 核心原因
Win32 API 默认的 `CreateFont` 品质参数较低，或者系统没有开启次像素平滑渲染。

### 解决方案
在 `CreateFontW` 时，明确指定字体品质为 **`CLEARTYPE_QUALITY`**。

```cpp
m_hFont = CreateFontW(
    hPixel, 0, 0, 0, FW_NORMAL, FALSE, FALSE, FALSE, 
    DEFAULT_CHARSET, OUT_DEFAULT_PRECIS, CLIP_DEFAULT_PRECIS, 
    CLEARTYPE_QUALITY, // 关键：开启 ClearType 渲染
    DEFAULT_PITCH | FF_DONTCARE, 
    L"Microsoft YaHei" // 推荐使用微软雅黑
);
```

## 2. 字体模糊问题 (Blurry Scaling)

### 现象描述
在高分屏（如 4K 或 150% 缩放的笔记本）上，除了状态栏外，主界面的文字和按钮边框像被抹了奶油一样模糊。

### 核心原因
**DPI 虚拟化 (DPI Virtualization)**：当程序未声明高清感知时，Windows 会将程序绘制在 96 DPI 的虚拟画布上，然后通过位图拉伸的方式将其放大。

### 解决方案
必须实现 **Per-Monitor V2** 级别的 DPI 感知。

#### A. 资源清单 (Manifest) 注入 (最可靠)
在 `app.manifest` 中声明高清感知，并必须在 `.rc` 脚本中显式引用。
```xml
<asmv3:application>
    <asmv3:windowsSettings>
        <dpiAware xmlns="http://schemas.microsoft.com/SMI/2005/WindowsSettings">true/pm</dpiAware>
        <dpiAwareness xmlns="http://schemas.microsoft.com/SMI/2016/WindowsSettings">PerMonitorV2</dpiAwareness>
    </asmv3:windowsSettings>
</asmv3:application>
```

#### B. 动态加载 shcore.dll
在 `WinMain` 启动瞬间强制开启感知，确保逻辑层不被系统接管。

## 3. 局部清晰与模糊的“初始化陷阱”

### 现象描述
左下角的状态栏清晰，但中间的按钮和输入框始终无法变清晰。

### 核心原因
**初始化顺序错误**：在调用 `EnumChildWindows` 设置字体时，字体对象 (`m_hFont`) 尚未创建完成（为 `NULL`）。控件会回退到系统默认的低分辨率点阵字体，然后再被 DPI 缩放强行拉伸。

### 修正逻辑
1.  **获取 DPI**：`GetDpiForWindow`。
2.  **创建字体**：基于 DPI 计算物理高度。
3.  **创建控件**：控件创建后立即通过 `WM_SETFONT` 应用已生成的字体句柄。

---

## 4. 物理像素匹配算法 (Pixel Snapping)

### 计算公式
不能直接写死 18px。必须根据 DPI 动态计算：
$$Height = -MulDiv(PointSize, DPI, 72)$$

### 布局缩放
所有坐标 $(x, y)$ 和尺寸 $(w, h)$ 必须经过缩放函数处理：
```cpp
int Scale(int value) {
    return (int)(value * m_scale + 0.5f); // 引入 0.5f 四舍五入，确保像素对齐
}
```

---

## 5. 总结

| 优化前 (Ugly) | 优化后 (Sharp) | 关键技术项 |
| :--- | :--- | :--- |
| `ANTIALIASED_QUALITY` | `CLEARTYPE_QUALITY` | 边缘锐化 |
| 默认 DPI Aware | PerMonitorV2 Manifest | 拒绝位图拉伸 |
| 硬编码像素 | MulDiv(pt, dpi, 72) | 动态高清适配 |
| 后置生成字体 | 先置创建字体对象 | 句柄有效载入 |

---
> [!TIP]
> **最佳实践**：始终将 Manifest 文件设为 `1 RT_MANIFEST` 类型嵌入资源字段，并在任何窗体操作前通过 `SetProcessDpiAwarenessContext` 进行环境隔离。
