# URL 与 GitHub 安装指令逻辑优化计划

目前系统在处理 URL 或 GitHub 仓库路径时存在缺陷：
1. **URL 被误当作包名**：生成的 `if` 判断语句中，`requireNamespace` 和 `packageVersion` 错误地填充了完整的 URL 字符串。
2. **全局版本误导**：在 URL 安装模式下，全局版本号被强行加入判断，导致生成的 R 代码逻辑错误且无法执行。

## 方案设计

### 1. 完善 GenerateCommand 逻辑 (pkg_logic.cpp)
- **增加 URL 检测**：检测 `url` 参数是否以 `http://` 或 `https://` 开头。
- **URL 模式短路**：如果检测到是 URL，且使用的是 `Remotes` (install_url) 或 `Devtools` 模式，直接跳过 `if (conditional)` 包装逻辑，只返回原始安装指令。这样可以避免无效的包名检测和版本检测。
- **GitHub 模式优化**：
    - 在生成 `remotes::install_github` 指令时，从 `user/repo` 中提取 `repo` 作为 `pkgName` 用于条件判断。
    - 如果提取失败，则回退到不使用 `if` 包装。

### 2. 增强包名提取算法 (pkg_logic.cpp)
- 优化 `ExtractPackageName` 功能，使其能处理更多边缘情况，或者在 `GenerateCommand` 中根据 `method` 直接指派 `pkgName`。

## 验证计划
### 场景 1：输入 GitHub 归档 URL
- 输入：`https://github.com/MRCIEU/gwasglue/archive/refs/tags/0.0.0.9000.tar.gz`
- 启用“条件安装模式”和“全局版本”。
- **预期结果**：仅生成 `remotes::install_url(...)`，不带 `if` 包装。

### 场景 2：输入 GitHub 仓库路径
- 输入：`MRCIEU/gwasglue`
- 选中 `GitHub (remotes)` 模式。
- **预期结果**：
  ```r
  if (!requireNamespace("gwasglue", quietly = TRUE)) {
    remotes::install_github("MRCIEU/gwasglue", upgrade = "never")
  }
  ```
