### 根因

项目复杂度主要来自以下工程因素：

- 应用同时承担离线脚本生成与联网多源检索，前后端职责存在强交互关系。

## 核心发现

1. 当前有效主工程为 `mod_UI`，不是根目录下的历史 Python、JavaScript 构建脚本或 `release` 产物。
2. 应用架构清晰分为 React 前端编排层与 Rust/Tauri 命令层。前端负责交互、状态、视图和本地轻量持久化；后端负责输入解析、脚本生成、网络检索、配置文件、缓存、历史记录和敏感信息保护。
3. `App.tsx` 是前端总控组件，集中管理视图切换、输入、方法选择、脚本生成、检索启动、历史应用、主题字体和快捷键。
4. `useSearch` 封装检索任务生命周期，负责启动、停止、进度事件订阅、日志接收和报告页切换。
5. `useSettings` 封装配置加载、保存、Token 状态、代理、镜像、缓存、依赖解析和全量检索等设置。
6. `useHistory` 封装历史命令加载、清洗、复制、删除、清空和异步保存。
7. Rust 后端的 `logic.rs` 是安装脚本生成核心，依据输入类型、检索结果、版本、镜像和安装策略生成 R 命令。
8. Rust 后端的 `search.rs` 是多源检索核心，包含输入过滤、缓存命中、HTTP 客户端构建、批量并发、进度事件、日志事件、超时控制、取消控制和依赖图触发。
9. `storage.rs` 承担应用数据目录下 `settings.json`、`history.json`、缓存和输入过滤规则的读写，并具备损坏 JSON 备份回退能力。
10. `secrets.rs` 使用 Windows DPAPI 保护 GitHub Token，前端 `load_settings` 不回传 Token 明文，符合 README 中声明的安全边界。
11. `tauri.conf.json` 启用了生产 CSP、原型冻结、无痕窗口、禁用拖放和固定更新端点，前端连接能力被限制在 IPC 与本地开发服务范围内。
12. CI 已覆盖版本一致性检查、前端测试、Rust 格式检查、Clippy、Rust 测试和 Tauri 无打包构建。
