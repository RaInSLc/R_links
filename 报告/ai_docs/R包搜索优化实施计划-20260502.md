# R 包搜索逻辑优化实施计划

旨在解决 `FigR`, `Pando`, `Seqinfo`, `BSgenome.Mmusculus.UCSC.mm10` 等包无法在工具中搜到的问题。

## 用户评审要求
> [!IMPORTANT]
> 本次修改涉及网络请求逻辑和 HTML 文本解析。如果网络代理配置不当，GitHub API 可能会因频率限制（Rate Limit）导致搜索失败。建议用户在 `config.ini` 中配置有效的 GitHub Token 以获得更高的限额。

## 方案要点

### 1. Bioconductor 检索鲁棒性提升
- **问题**：目前的 HTML 匹配 `<td>Version</td>` 太过死板，容易被空格或换行符干扰。
- **改进**：
    - 采用更宽松的匹配逻辑，忽略 `<td>` 和 `Version` 之间的空白。
    - 确保完整读取大型页面（如 BSgenome 系列），目前的 512KB 缓冲区已足够，但需确保读取循环逻辑健壮。

### 2. GitHub 搜索精细化过滤
- **问题**：`Pando` 容易被 `pandoc` 等高星 R 包干扰（`pandoc` 包含 `pando`）。
- **改进**：
    - 在评分逻辑中，将“包含匹配”的权重降低，仅在没有完全匹配时作为候选。
    - 强制执行反向验证：下载 `DESCRIPTION` 文件并检查 `Package:` 字段，必须与输入包名**完全一致**（忽略大小写）。

### 3. r-universe 优先级与解析优化
- **问题**：`Seqinfo` 等 Bioconductor 包现在也托管在 GitHub/r-universe 上。
- **改进**：
    - 优化 JSON 解析，确保能正确处理 `r-universe` 返回的数组格式。
    - 提取 `RemoteUrl` 以自动获取其 GitHub 仓库地址，便于后续生成 `install_github` 命令。
