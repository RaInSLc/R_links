# 解决 Auto 与 packageVersion 逻辑逻辑解耦计划

目前 `Auto` 模式和 `packageVersion` 按钮在生成带有版本号的脚本时逻辑发生了混淆：
- `packageVersion` 按钮现在正确生成检测语句，但 `Auto` 模式由于错误路由到了该模式，也生成了检测语句。
- 实际上，`Auto` 模式在识别到版本号时，应该生成**安装指令**（`install_version`），而不是检测指令。

## 方案设计

### 1. 扩充安装方法枚举 (pkg_logic.hpp)
在 `InstallMethod` 枚举中添加内部专用的渲染方法：
- `RemotesVersion`: 专门用于生成 `remotes::install_version(...)` 安装指令。

### 2. 重塑渲染逻辑 (pkg_logic.cpp)
- **InstallMethod::Version**: 保持为 `packageVersion("包名")`，用于纯粹的版本检测。
- **InstallMethod::RemotesVersion**: 实现为 `remotes::install_version("包名", version="...", ...)`。且该方法**支持**条件安装包装（`if` 块）。

### 3. 修正智能路由逻辑 (app_window.cpp)
- 修改 `UpdateGeneratedCommand` 中的“智能增强”部分。
- 当 `m_currentMethod == Auto` 且版本号存在时，将 `effectiveMethod` 提升为 `RemotesVersion`（即生成安装指令），而不是 `Version`（版本检测）。

## 验证计划
### 场景 1：手动选中 packageVersion
- 输入 `xfun 0.47`。
- 点击 `packageVersion` 按钮。
- **预期结果**：输出 `packageVersion("xfun")`。

### 场景 2：选中 Auto 模式并进行体检/手动输入版本
- 输入 `xfun 0.47`。
- 选中 `Auto` 模式。
- **预期结果**：输出包含 `remotes::install_version("xfun", version="0.47", ...)` 的 `if` 安装块。
