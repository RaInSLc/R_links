# cpp_src 检索稳定性与正确性重构实施方案

> 生成日期: 2026-06-07 | 类型: 方案 | 状态: 草稿

## 背景

参考对 `cpp_src` 的逻辑完整性审计报告，当前程序在停止控制流、异常状态提示、Bioconductor 映射推断公式以及 HTML 解析失败容错等方面存在可优化的空间。为了提高批量检索的可信度与工具稳定性，需要对这些点进行针对性重构。

## 实施方案

### 方案概述
1. **停止状态隔离与命令阻断**：在工作线程被停止时，向主线程投递带有标记的 `WM_SEARCH_COMPLETE`。主线程收到停止信号后只做状态更新，不再自动切换至 Auto 模式及重新生成预览指令，避免使用部分不完整的结果。
2. **初始化失败显式通知**：当 `InternetOpenW` 会话句柄打开失败时，主动投递通知消息，杜绝界面陷入无限“扫描中”的静默等待状态。
3. **HTML/JSON 解析容错**：在 Bioconductor 解析网页时，只有在成功提取出合法的版本号时才将 `anyFound` 置为 `true`，避免因 HTTP 200 解析空值而错误阻断后续的回退（如 GitHub 源）检索。
4. **Bioconductor 版本演进跟进**：
   - 在历史遍历版本列表中新增最新的 `3.23` 版本支持。
   - 修复 `InferBiocVersion` 推断公式中对主版本 1、次版本 34 至 48 映射公式存在的 10 个次版本偏置漏洞（公式由 `(pkgMinorVersion - 34) / 2` 修正为 `(pkgMinorVersion - 34) / 2 + 10`），使其精确映射至 3.10 至 3.17，大幅减少探测等待开销。

### 预期效果
- 用户手动点击“停止”后，界面状态栏能够准确给出“体检任务已手动停止”提示，并且预览文本框保持原状，不再依据部分不完整的结果重新拼装生成 R 指令。
- 网络断开或初始化失败时有明确的错误提示，不再卡死。
- 当上游 Bioconductor 解析失败或版本空置时，在快速模式下可以正确回退到 GitHub 进行查找。
- 输入类似于 `GSVA 1.34` 的旧包名，其 Bioconductor 版本能一步精准推断为 `3.10`（原公式误推断为 `3.0`），检索响应速度极大提升。
