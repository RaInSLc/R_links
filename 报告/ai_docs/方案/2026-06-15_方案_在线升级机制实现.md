# 在线升级方案实现总结

> 生成日期: 2026-06-15 | 类型: 方案 | 状态: 已实施

## 背景

为了方便用户平滑过渡到新版本、修复 Bug 并第一时间获取新特性，需要为应用加入在线自动升级功能。根据讨论及设计，采用了业界标准的 **Tauri Updater 结合 GitHub Actions & Releases** 流程。

## 方案分析与核心组件

### 工作原理
- **持续集成 (CI)**：借助于新建的 `.github/workflows/release.yml`，每次在 GitHub 仓库发布带有 `v*` 标签的代码推送时，GitHub Action 会自动启动 Tauri 构建进程。
- **发布资源托管 (CD)**：构建成功后，GitHub Action 利用 `tauri-apps/tauri-action` 自动发布一条 GitHub Release，并在 Release 中附加编译好的可执行程序以及一份 `latest.json`。
- **端侧升级组件**：在前端和后端引入了 Tauri Updater 插件。客户端通过向配置好的 GitHub 接口查询 `latest.json`，比对现有版本。如果发现更新，即可调用内部下载器覆盖安装并提示用户。

### 本次技术更改
1. **依赖添加**：在 `Cargo.toml` 增加 `tauri-plugin-updater` 及其权限声明，并在前端 `package.json` 引入对应 NPM 模块。
2. **后端注册与配置**：`lib.rs` 注册了 updater 插件。`tauri.conf.json` 中预埋了用于 GitHub Release 的拉取端点。
3. **前端 UI**：`App.tsx` 中的“连接设置”后方加入了一个独立的“应用更新”面板，支持手动触发检测、展示下载进度及安装完成的事件监听。
4. **Action 文件**：`.github/workflows/release.yml` 定义了用于自动执行上述打包和分发任务的声明。

目前基础代码已全面适配完毕，但因为涉及到私钥等敏感操作，尚需您进行以下配置才能完全跑通更新链路：

| 步骤 | 操作说明 | 负责模块 |
|------|------|----------|
| 1 | **修改 Endpoint**：在 `mod_UI/src-tauri/tauri.conf.json` 的 `plugins.updater.endpoints` 中，将 `USERNAME/REPO` 替换为您真实的 GitHub 用户名和仓库名。 | 研发人员 |

| 4 | **配置 GitHub Secrets**：将上一步生成的私钥和密码分别添加为 GitHub 仓库的 Secrets：`TAURI_SIGNING_PRIVATE_KEY` 与 `TAURI_SIGNING_PRIVATE_KEY_PASSWORD`。 | 仓库管理员 |
