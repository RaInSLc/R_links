# GitHub 权威检索集成计划 (最终版)

经过深度调研，我们确认 `r-universe` 和 `rdrr.io` 在处理冷门或处于活跃开发状态的 GitHub 包时存在索引滞后或遗漏的问题。本计划将直接集成 **GitHub 官方 Search API**，提供最权威、最实时的检索能力。

## 方案设计

### 1. 终极检索链路 (app_window.cpp)
将体检链路调整为：
**CRAN $\rightarrow$ Bioc $\rightarrow$ r-universe $\rightarrow$ GitHub API (新)**。

### 2. GitHub API 集成细节
- **API 请求**：访问 `https://api.github.com/search/repositories?q={pkg}+language:R&sort=stars`。
- **User-Agent 伪装**：GitHub API 强制要求 `User-Agent`。我们将为所有网络请求统一增加该请求头，以确保稳定性。
- **仓库锁定**：从 JSON 结果的 `items` 中提取第一项作为最匹配的仓库（通常是星标最高的官方或活跃版本）。
- **智能提取**：从 `full_name` 获取 `user/repo` 路径。

### 3. DESCRIPTION 解析优化
- **多分支兼容**：一次性探测 `master`、`main`、`devel` 等常用开发分支。
- **健壮性增强**：优化字符串提取逻辑，处理 R 包描述文件中可能出现的各种换行符和空格差异。

## 验证计划
### 专项攻坚测试
- **测试用例 1**：`SeuratDisk` -> 锁定 `mojaveazure/seurat-disk` 并抓取版本。
- **测试用例 2**：`irGSEA` -> 锁定 `chuiqin/irGSEA` 并抓取最新版本。

## 风险提示
> [!IMPORTANT]
> **API 频率限制**：GitHub 匿名搜索 API 有 1 分钟 60 次的限制（针对单个 IP）。对于普通个人体检列表（几十个包）完全够用。对于超大型列表（几百个包），搜索速度会自动受限以防被锁。
