# 检索降级逻辑优化计划 (严格版本校验版)

针对用户反馈的 `rsnps`、`ggDCA` 等包在 CRAN 只有占位页面却无版本号导致检索中断的问题，本计划将引入“版本存在性驱动”的降级机制。

## 方案设计

### 1. 严格版本校验逻辑 (app_window.cpp)
在 `SearchWorkerThread` 循环中：
- **CRAN/Bioc 检索块**：
    - 执行网页抓取后，检查版本号 `ver` 是否提取成功。
    - **核心逻辑**：如果无法解析出版本号（说明虽然 URL 存在但包已失效/移除），即使返回 200，也将 `found` 重置为 `false`。
    - 这将“诱导”程序跳过当前源，继续尝试后续的 Bioconductor 或 GitHub 检索，实现真正的自动穿透。

### 2. GitHub 检索关键词优化
- **召回率提升**：针对 `SeuratDisk` 容易被漏掉的问题，优化 GitHub Search API 的搜索权重，优先匹配仓库名。

## 验证计划
### 针对性回测
- **rsnps**：应能穿透 CRAN 占位符，最终定位到 GitHub 真实版本。
- **ggDCA**：应能穿透 CRAN，找回其他源中的版本。
- **SeuratDisk**：应能精准找回 `mojaveazure/seurat-disk`。
