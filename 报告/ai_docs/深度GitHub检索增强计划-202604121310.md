# 深度 GitHub 检索增强计划

目前的 r-universe 检索对于某些未全球索引的包（如 `SeuratDisk`）会失效。本计划通过引入 `rdrr.io` 的重定向路由和 GitHub 原生描述文件解析，补齐最后一块短板。

## 方案设计

### 1. 检索链路四级化 (app_window.cpp)
在 `SearchWorkerThread` 中增加第四梯度检索：
- **顺序**：CRAN $\rightarrow$ Bioc $\rightarrow$ r-universe $\rightarrow$ **rdrr.io Fallback**。
- **实现逻辑**：
    - 访问 `https://rdrr.io/find/?q={pkg}`。
    - 利用 `WinInet` 自动跟踪重定向。
    - 检查最终跳转的 URL。如果包含 `/github/{user}/{repo}/`，则判定检索成功。

### 2. GitHub 原生版本采集 (app_window.cpp)
一旦通过 `rdrr.io` 锁定了 GitHub 仓库路径：
- **路径提取**：从跳转后的 URL 提取 `user/repo` 信息。
- **版本抓取**：尝试访问 `https://raw.githubusercontent.com/{user}/{repo}/master/DESCRIPTION`（或 `main` 分支）。
- **解析逻辑**：从 DESCRIPTION 文件中精准提取 `Version: x.y.z`。

### 3. 数据稳定性处理
- 针对 GitHub 的 API 频率限制（或 raw 内容访问）进行异常处理。
- 如果无法抓取到版本号，则仅显示 `Found (GitHub)` 而不显示具体版本，以保证流程不阻塞。

## 验证计划
### 手动测试
- **测试用例**：输入 `SeuratDisk`。
- **预期流程**：
    1. CRAN/Bioc/r-universe 均返回未找到。
    2. rdrr.io 命中并重定向至 `mojaveazure/seurat-disk`。
    3. 获取版本号（如 `0.0.0.9001`）。
- **预期结果**：
    - 报告显示：`✅ [GitHub] SeuratDisk (v0.0.0.9001) -> mojaveazure/seurat-disk`。
    - Auto 路由生成：`remotes::install_github("mojaveazure/seurat-disk")`。
