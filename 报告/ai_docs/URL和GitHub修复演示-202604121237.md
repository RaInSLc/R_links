# URL 与 GitHub 安装指令优化演示 (2026-04-12)

本次更新解决了在粘贴 URL 时生成脚本逻辑混乱的问题。系统现在变得更加“聪明”，能够识别复杂的链接结构。

## 改进要点

### 1. 智能包名提取
系统现在可以从各种格式的链接中提取真正的 R 包名：
- **GitHub 归档**：`https://github.com/user/pkg/archive/...` $\rightarrow$ `pkg`
- **GitHub 仓库**：`user/pkg` $\rightarrow$ `pkg`
- **通用 URL**：`https://.../pkg_1.2.3.tar.gz` $\rightarrow$ `pkg`

### 2. 逻辑短路保护
> [!TIP]
> **全局版本屏蔽**：当用户选择基于 URL 的安装（如 `install_url` 或 `install_github`）时，系统会自动忽略 UI 中的“全局版本号”。这是因为第三方链接通常代表了特定的版本，强行使用全局版本号进行 `packageVersion()` 比较往往是错误的。

### 3. 条件安装增强
即使是 URL 模式，现在也会生成 `if (!requireNamespace("包名", ...))` 块，实现了您要求的“不跳过 if”且“包名正确”。

## 渲染对比示例

### 输入
- **列表内容**：`https://github.com/MRCIEU/gwasglue/archive/refs/tags/0.0.0.9000.tar.gz`
- **全局版本**：`1.1.1`（将被智能忽略）
- **选项**：启用“条件安装模式”

### 修改前（错误）
```r
if (!requireNamespace("https://github.com/...", ...) || packageVersion("https://...") != "1.1.1") {
  remotes::install_url("https://github.com/...")
}
```

### 修改后（正确）
```r
if (!requireNamespace("gwasglue", quietly = TRUE)) {
  remotes::install_url("https://github.com/MRCIEU/gwasglue/archive/refs/tags/0.0.0.9000.tar.gz")
}
```
