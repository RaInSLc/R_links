# UI 交互逻辑修复报告 - 202604101452

## 1. 现象描述
用户发现体检结束后，界面会自动跳转到“Check System”单选按钮上。虽然体检结果已出，但用户更希望系统能回到“Auto”模式，以便利用体检获取的版本信息生成“智能路由”脚本。

## 2. 修复细节
定位到 `app_window.cpp` 中的 `WM_SEARCH_COMPLETE` 消息处理函数。该函数在后台搜索线程结束后被触发。

**修改前代码**：
```cpp
CheckRadioButton(hw, IDC_RADIO_DEVTOOLS, IDC_RADIO_CHECK, IDC_RADIO_CHECK);
p->m_currentMethod = InstallMethod::CheckSystem;
```

**修改后代码**：
```cpp
// 自动切回 Auto 模式
CheckRadioButton(hw, IDC_RADIO_DEVTOOLS, IDC_RADIO_CHECK, IDC_RADIO_AUTO);
p->m_currentMethod = InstallMethod::Auto;
```

这样做的好处是，体检一结束，系统就会立刻根据刚拿到的 CRAN/Bioc 版本信息，配合“Auto”模式的智能算法，直接渲染出最精准的安装脚本。

## 3. 编译情况
已解决编译时的文件占用问题（Permission denied），成功生成最新的 `RLinkInstaller_PRO.exe`。
