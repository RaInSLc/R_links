# UI 动态屏蔽与深度生成逻辑重构计划

根据您的需求，在识别到粘贴内容为链接（URL）时，界面本身应当更“智能”地拒绝不合适的操作，并且底层的生成逻辑也要对链接输入予以放宽，不让提取包名的正则去破坏一段完好的网址。

## 方案设计

### 1. 动态 UI 交互 (app_window.cpp)
在每次 `EN_CHANGE` (文本框内容改变触发实时刷新) 时：
- 检测输入字符串中是否包含 `http://` 或 `https://`。
- **如果是链接**：
    - **禁用 (置灰)** 此前用于包名的 6 个单选按钮：`GitHub (remotes)`, `BiocManager`, `install.packages`, `packageVersion`, `Auto`, `Check System`。
    - **强制重定向**：如果用户当前正好处在这 6 个模式之一，自动帮用户勾选 `remotes::install_url`。
- **如果不是链接**：
    - 恢复这 6 个单选按钮的启用状态。

### 2. 底层逻辑深度剥离 (pkg_logic.cpp)
深度检测当前的正则解析链：
- 现在的 `PkgLogic::ExtractPackageAndVersion` 有一行代码：`std::regex_replace(input, ..., L" ")`，它会野蛮地把您粘贴进去的 `http://xxx` 删掉，导致后续传给生成器的永远是一个空包名。
- 这引发了严重冲突。
- **对策**：如果当前的 `m_currentMethod` 是 `Devtools` 或 `Remotes` (代表此时必然是在安装 URL)，我们就在循环体内**绕过**负责清理杂音的 `ExtractPackageAndVersion` 函数，直接将整行文本视为 `p` 传递给最终的生成核心，这样 URL 的完全性得以保留。

1. 您粘贴 `survdistr` 时，界面还是现在的日常界面，所有按钮可用。
