# 脚本生成精准度与垃圾字符清理计划

目前系统在解析用户输入的包列表时，会误将行尾的注释、网址和说明文字识别为版本号，导致生成类似 `version = "0.6.1 版本不存在..."` 的错误代码。本计划旨在通过增强解析逻辑和源同步机制，彻底解决此问题。

## 方案设计

### 1. 精准版本提取 (pkg_logic.cpp)
- **正则化清洗**：修改 `ExtractPackageAndVersion`，引入 `<regex>` 支持。
- **匹配逻辑**：
    - **预处理**：首先移除行内所有的 URL（以 `http` 开头的部分）。
    - **精确提取**：匹配第一个类似 `0.6.1` 或 `1.4-2` 的数字序列作为版本号。
    - **彻底隔离**：版本号之后的任何中文说明或非版本字符都将被丢弃。

### 2. 体检结果权重升级 (app_window.cpp)
- **智能源切换**：在 `UpdateGeneratedCommand` 中，如果“全量体检报告”已经明确了该包的来源（如 GitHub 或 Bioc）：
    - **强制纠偏**：即使原输入带有垃圾版本信息，生成的指令也将强制使用体检发现的来源（如 `install_github`）。
    - **状态同步**：当体检锁定 GitHub 时，将自动忽略用户输入中可能引发冲突的 `v` 参数，优先保证安装成功。

### 3. IOBR & SeuratDisk 专项修正
- 修复因输入包含“github”字样导致生成引擎逻辑混乱的问题，确保生成的代码只包含 R 运行时能识别的纯净参数。

## 验证计划
### 场景回测
- **输入**：`rsnps 0.6.1 版本不存在 https://github.com/ropensci/rsnps`
- **预期输出**：生成的 R 脚本中 `version` 字段应仅为 `"0.6.1"` 或直接切换为 `install_github` 模式。
