# 网络增强与配置扩展计划 (代理 & GitHub Token)

为了提升在复杂网络环境（如国内网络）下的稳定性，并解决 GitHub API 的匿名频率限制，本计划将为软件引入代理支持和 Token 认证功能。

## 用户评审项
- **配置文件**：设置将保存在程序目录下的 `config.ini` 中。
- **代理格式**：支持标准 HTTP 代理格式（如 `127.0.0.1:7890`）。
- **隐私保护**：GitHub Token 在界面上将以掩码（密码模式）显示。

## 方案设计

### 1. UI 界面调整 (app_window.cpp)
- **新增选项卡**：在 `TabCtrl` 中增加第四个选项卡 “Settings (设置)”。
- **新增控件**：
    - `IDC_ENTRY_PROXY`：编辑框，用于输入代理地址。
    - `IDC_ENTRY_TOKEN`：编辑框，用于输入 GitHub Token (ES_PASSWORD)。
    - `IDC_BTN_SAVE_CONFIG`：保存按钮。

### 2. 配置持久化逻辑 (app_window.cpp)
- 使用 Windows API `GetPrivateProfileString` 和 `WritePrivateProfileString` 实现对 `config.ini` 的读写。
- 程序启动时加载配置；点击保存按钮时更新文件并同步到内存变量。

### 3. 网络逻辑增强 (app_window.cpp)
- **代理注入**：在 `SearchWorkerThread` 中，如果配置了代理，将 `InternetOpenW` 的打开类型切换为 `INTERNET_OPEN_TYPE_PROXY` 并传入代理字符串。
- **Token 认证**：在所有 GitHub API 和 Raw 内容请求中，如果配置了 Token，额外注入 `Authorization: token {YOUR_TOKEN}` 头部。

## 验证计划
### 手动测试
- **代理验证**：配置一个错误的代理，验证体检是否报错；配置正确的代理，验证下载是否加速。
- **Token 验证**：配置有效 Token 确认 GitHub API 是否能正常检索。
