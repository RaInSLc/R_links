# 镜像联动与保存逻辑深度诊断修复计划 (彻底解决无效问题)

针对“下拉框选择后不填充”以及“保存不生效”这两个核心顽疾，我们将通过“全透明化追踪”和“底层写盘重构”来彻底根治。

## 方案设计

### 1. 引入消息调试追踪 (诊断层)
- **状态栏实时回显**：我将修改 `OnCommand` 逻辑，在底层增加调试输出。当您操作下拉框或点击保存时，底部的状态栏会实时显示捕捉到的 ID 和消息码。
- **确诊问题点**：如果点击下拉框后状态栏没反应，说明 Win32 消息可能被屏蔽了；如果有反应但没变，则是 URL 字符串处理逻辑有误。

### 2. 重构联动同步逻辑 (操作层)
- **多事件复合监听**：同时监听 `CBN_SELCHANGE`（改变时）、`CBN_SELENDOK`（确认时）和 `CBN_CLOSEUP`（下拉框收起时）三个节点。只要您选了，就必须同步。
- **控件句柄强校验**：不再依赖不稳定的成员变量，每次同步都强制使用 ID 索引句柄。

### 3. 持久化层推倒重来 (持久化层)
- **弃用 Win32 旧 API**：彻底抛弃带有黑盒缓存机制的 `WritePrivateProfileStringW`。
- **改用 C++ 文件流直接写盘**：换用 `std::wofstream` 这种“硬核”写入方式。写入即落盘，没有任何中间层缓存，确保您的 IDE 瞬间变色。

## 验证计划
- **操作反馈**：点击“R-Forge”后，状态栏应显示“同步成功”。
- **文件验证**：点击保存后，状态栏显示“写入成功：[N] 字节”。此时刷新 IDE 里的文件，内容应 100% 同步。
