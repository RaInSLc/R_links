# GitHub 检索集成与智能路由演示 (2026-04-12)

我们现在已经完成了包检索链路的“终极进化”，实现了 **CRAN $\rightarrow$ Bioc $\rightarrow$ GitHub** 的全自动体检覆盖。

## 核心功能提升

### 1. 三级全自动体检
体检系统现在会按以下优先级顺序自动穿梭在不同源之间：
- **第一级**：CRAN 官方标准库
- **第二级**：Bioconductor 生物信息学专库
- **第三级**：**GitHub (通过 r-universe API 检索)**
  - *这是本次更新的重点。即使您只输入一个包名（如 `gwasglue`），程序也会在 GitHub 海量的代码库中找到它的“真身”。*

### 2. 智能仓库路由
当体检发现某个包仅托管在 GitHub 上时，“Auto”模式将发挥前所未有的智能：
- **自动补全路径**：它会自动将您输入的简写包名（如 `gwasglue`）扩展为完整的仓库路径（如 `MRCIEU/gwasglue`）。
- **自动切换指令**：它会自动放弃 `install.packages`，转而生成 `remotes::install_github("user/repo")`。

## 操作演示

### 输入示例
1. 在输入框粘贴包名：`gwasglue`
2. 点击 **“开始全量异步体检”**

### 体检反馈
右侧报告区将实时滚动显示：
`✅ [GitHub] gwasglue (v0.0.0.9000) -> MRCIEU/gwasglue`

### 生成结果 (Auto 模式)
```r
# [GitHub Verified: v0.0.0.9000 | Auto-Synced]
if (!requireNamespace("gwasglue", quietly = TRUE)) {
  remotes::install_github("MRCIEU/gwasglue", upgrade = "never")
}
```
