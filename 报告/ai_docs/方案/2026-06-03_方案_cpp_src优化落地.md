# cpp_src 代码优化与稳定性增强实施方案

针对 `cpp_src` 经过多轮功能叠加后遗留的版本推断错误、句柄竞争风险、网络请求无超时及代码结构优化问题，本方案提出一整套具体落地优化设计。

## 背景

在对 `cpp_src` 的最新工作区代码以及相关评审文档（`2026-06-03_评审_cpp_src优化后增强与技术选型.md` 和 `2026-06-03_评审_BiocGit实现优化清单.md`）进行深入分析后，确认存在以下能够即刻落地优化的确定性与稳定性问题：

1. **版本推断公式错误**：原有 `InferBiocVersion` 仅使用次版本号无法推断主版本号升级（如 `2.0.x`）的情况，导致推断版本失效而走历史遍历。
2. **WinInet 句柄竞争风险**：UI 线程在停止/关闭时关闭 `m_hInternet`，工作线程同时持有并在完成时再次读取和关闭，缺少互斥锁或原子操作，存在句柄双重释放的隐患。
3. **缺少超时设置**：WinInet 的 `InternetOpenUrlW` 和 `InternetReadFile` 容易在网络状况不佳时长时间阻塞。
4. **异常路径句柄泄漏**：`SearchWorkerThread` 异常处理吞掉异常后，未确保局部句柄统一释放。
5. **GitHub fallback 弱化**：找到任一源即跳过的策略使得多源同时展示能力缺失。
6. **重试机制 goto 破坏可读性**：包名大小写纠正使用 goto 重试破坏了局部作用域。

## 详细改造设计

### 1. 修正版本推断规则（`pkg_logic.hpp/cpp`）
- 增加 `ParseMajorVersion` 静态方法。
- 重构 `InferBiocVersion` 接收 `(pkgMajorVersion, pkgMinorVersion)` 进行联合推算：
  - 包主版本为 `1`：偶数次版本号 `>= 50` 时，Bioc次版本号 = `(pkgMinorVersion - 50) / 2 + 18`；偶数次版本号 `34 <= pkgMinorVersion < 50` 时，Bioc次版本号 = `(pkgMinorVersion - 34) / 2`。
  - 包主版本为 `2`：偶数次版本号 `>= 0` 时，Bioc次版本号 = `pkgMinorVersion / 2 + 21`。
- 修改 `app_window.cpp` 对该公式的调用，直接拼出 `L"3." + std::to_wstring(biocVer)`。

### 2. 线程安全句柄管理与超时设置（`app_window.hpp/cpp`）
- 使用 `std::atomic<HINTERNET> m_hInternet{NULL}` 和 `std::atomic<HINTERNET> m_hActiveRequest{NULL}` 替换原有的裸句柄，确保跨线程可见性。
- 引入 RAII 辅助类 `SafeRequestHandle` 管理工作线程中的临时句柄，保障任意退出路径下句柄不泄漏。
- 在 `OnCommand` 手动停止与 `WM_DESTROY` 消息处理中，采用原子交换方式关闭句柄，杜绝双重释放：
  ```cpp
  m_stopRequested = true;
  HINTERNET hReq = m_hActiveRequest.exchange(NULL);
  if (hReq) InternetCloseHandle(hReq);
  HINTERNET hInt = m_hInternet.exchange(NULL);
  if (hInt) InternetCloseHandle(hInt);
  ```
- 为 `InternetOpenW` 返回的主句柄配置 30 秒的连接、发送、接收超时属性，使网络请求超时可控。

### 3. 快速模式/全量检索模式切换（`resources\resource.h`、`app_window.hpp/cpp`）
- 新增 `IDC_CHECK_FULL_SEARCH` (221) 控件。
- 在 Workspace 页面下方添加“全量检索所有源 (包括 GitHub)”的 Checkbox。
- 配置快照结构体 `ConfigSnapshot` 新增 `bool fullSearch` 属性。在快速模式下，CRAN/Bioc 命中后即跳过 GitHub 搜索；全量模式下则依然保留对 GitHub 渠道的检索与多源展示。

### 4. goto 重试消除与异常完善
- 将大小写纠正重试 `goto RETRY_SEARCH` 改写为 `while (retrySearch)` 循环状态机，保证局部作用域整洁。
- 完善 `SearchWorkerThread` 外层的 `catch` 块，输出异常诊断日志，并确保在发生异常时也能通过原子交换正确关闭 `m_hInternet`。

### 5. 非破坏性构建测试脚本（`build_test.bat`）
- 编写非破坏性编译脚本，输出 `RLinkInstaller_PRO_test.exe`，验证通过后自动删除临时文件与测试 exe，不带 `pause` 阻碍，适合自动化流程。

### 1. 离线测试
- 创建 `报告/ai_codes/bioc_infer_test.cpp` 包含各种包版本（1.50.x、1.52.x、2.0.x 等）到 Bioconductor 对应版本的推断，并利用 `build_test.bat` 验证编译。

- 体检扫描期间点击停止按钮，检查响应时效性及后续开始扫描的连贯性。
- 切换“全量检索模式”复选框，观察 GitHub 检索源是否被按需调用。
