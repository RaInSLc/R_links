# 脚本生成格式与版本逻辑修复报告 - 202604101448

## 1. 问题复现与分析
在之前的版本中，当用户启用“全量体检”并获取到版本号后，生成脚本存在以下偏差：
- **格式杂乱**：输出首行出现了非预期的包名及版本字符串。
- **逻辑降级**：尽管有版本号，但生成的安装命令仍为 `install_url`（URL 模式），导致安装时无法精确匹配版本。
- **包装缺失**：版本安装模式（`Version`）下，脚本无法被 `if (!requireNamespace...)` 结构包裹，导致重复安装。

## 2. 核心修复逻辑

### 2.1 智能路由算法升级
在 `app_window.cpp` 的 `UpdateGeneratedCommand` 中，我引入了**版本感知路由**：
```cpp
// 如果检测到版本号，且当前是 Auto 或 Remotes 模式
if (!targetVer.empty() && (effectiveMethod == InstallMethod::Auto || effectiveMethod == InstallMethod::Remotes)) {
    effectiveMethod = InstallMethod::Version; // 强制提升至版本安装模式
}
```
这确保了只要有版本信息存在，生成的命令必然是 `remotes::install_version`。

### 2.2 解除包装封禁
在 `pkg_logic.cpp` 中，我移除了对 `InstallMethod::Version` 参与 `conditional` 逻辑的硬编码限制：
```cpp
// 修改前
if (conditional && !raw_cmd.empty() && method != InstallMethod::Version)
// 修改后
if (conditional && !raw_cmd.empty())
```
