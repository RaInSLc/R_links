# 字体切换功能开发总结

> 生成日期: 2026-06-14 | 类型: 总结 | 状态: 已确认

## 背景

在之前强制全应用升级为现代 Web 字体 (Inter + Noto Sans SC) 后，极大地改善了由于底层渲染引擎导致的中文小字号发糊和边缘锯齿问题。基于用户反馈，进一步增加在设置界面的“字体切换”控制开关，使用户能够自由地在高质量现代字体与 Windows 传统原生字体间进行选择。

## 实施方案

### 方案概述
采用基于 HTML 自定义数据属性（`data-font`）与全局 CSS 变量（CSS Variables）映射结合的架构，辅以 React 的 State 注入和 LocalStorage 存储。

| 步骤 | 操作 | 负责模块 | 风险等级 |
|------|------|----------|----------|
| 1    | 在 `App.css` 顶部基于 `[data-font]` 定义 `--font-base` 和 `--font-mono` 变量组，支持三种风格：“现代”、“系统”、“传统” | 前端样式 (mod_UI) | 低 |
| 2    | 使用 Node.js 脚本静态替换 CSS 中一百多处硬编码的 `font-family` 声明为 CSS 变量引用 | 前端样式 (mod_UI) | 低 |
| 3    | 在 `App.tsx` 引入 `currentFont` 全局状态以及相应的 `handleFontChange` 回调函数，并在底层写入 LocalStorage 进行长期记忆 | 前端状态 (mod_UI) | 低 |

### 预期效果
- 当用户进入“网络设置”并在“显示与外观”区块点击不同字体时，底层 DOM 根节点的 `data-font` 被立刻改写，触发浏览器全局样式的极速重绘，实现无刷新秒换字体。
- 程序下次启动时会自动读取记忆配置，并还原用户最偏爱的呈现效果。
