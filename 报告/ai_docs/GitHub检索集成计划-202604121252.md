# GitHub 集成体检与智能路由计划

目前系统的“全量异步体检”仅支持 CRAN 和 Bioconductor。本计划旨在将 GitHub 纳入体检范围，实现真正意义上的“全引擎”检索。

## 方案设计

### 1. 检索链路增强 (app_window.cpp)
在 `SearchThread` 函数中增加第三梯度检索：
- **顺序**：CRAN $\rightarrow$ Bioc $\rightarrow$ **GitHub (via r-universe)**。
- **GitHub 检索实现**：
    - API 地址：`https://r-universe.dev/api/search?q=Package:{pkg}&limit=1`。
    - 该接口由 r-universe 提供，索引了几乎所有在 GitHub 上公开的 R 包。
- **数据提取**：
    - 从 JSON 返回结果中匹配 `"Version": "..."` 和 `"RemoteUrl": "..."`。
    - 将发现的 GitHub 上游仓库信息存入检索结果。

### 2. 体检结果结构升级 (app_window.hpp)
- 更新 `SearchResult` 结构体，增加 `remoteUrl` 字段用于存储发现的 GitHub 仓库路径（如 `MRCIEU/gwasglue`）。
- 增加一个新的来源类型（例如 `source = 2` 代表 GitHub）。

### 3. 智能路由适配 (app_window.cpp)
在 `UpdateGeneratedCommand` 逻辑中：
- 如果体检结果标记为 `GitHub`：
    - 在 `Auto` 模式下，强制提升 `effectiveMethod` 为 `InstallMethod::Github`。
    - 自动将输入框中的“包名”替换/增强为体检发现的“user/repo”路径，确保生成正确的 `remotes::install_github("user/repo")` 脚本。

## 验证计划
### 手动验证
- **测试用例**：输入 `gwasglue`（已知仅存在于 GitHub）。
- **流程**：点击“开始全量异步体检”。
- **预期结果**：
    1. 状态栏/提示显示：`[GitHub Verified: v0.0.0.9000 | Auto-Synced]`。
    2. 生成预览显示：
       ```r
       if (!requireNamespace("gwasglue", quietly = TRUE)) {
         remotes::install_github("MRCIEU/gwasglue", upgrade = "never")
       }
       ```
