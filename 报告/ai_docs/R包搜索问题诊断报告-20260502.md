# R 包搜索失败问题诊断报告 - 20260502

## 1. 问题描述
用户反馈在 `RLinkInstaller` 中搜索 `Pando`, `FigR`, `Seqinfo`, `BSgenome.Mmusculus.UCSC.mm10` 等包时经常失败，提示找不到包或无法定位安装源。

## 2. 诊断结果
经过对代码审计和网络探测，发现以下核心瓶颈：

### 2.1 Bioconductor 解析截断
- **现象**: `BSgenome.Mmusculus.UCSC.mm10` 等包在 Bioconductor 官方页面上存在，但工具报告未找到。
- **原因**: `app_window.cpp` 中的 `InternetReadFile` 仅读取了前 8192 字节（8KB）。Bioconductor 的 HTML 页面通常较大（如 45KB+），版本信息 `<td>Version</td>` 位于页面中后部，被截断导致无法匹配。

### 2.2 GitHub 搜索噪音大
- **现象**: 搜索 `Pando` 时，GitHub 返回大量非 R 语言项目（如 `pandoc`），且由于其 stars 较高，占据了搜索结果前列。
- **原因**: 现有的搜索查询 `q=Pando` 过于宽泛。R 包通常在 GitHub 的 R 语言分类下，且开发者通常会给仓库打上 `r-package` 标签。

### 2.3 包名解析正则限制
- **现象**: 部分带有点号的包名提取不全。
- **原因**: `pkg_logic.cpp` 中的正则逻辑需要适配更复杂的 R 包命名规范。

## 3. 修复方案
1. **缓冲区优化**: 在 `app_window.cpp` 中实现循环读取逻辑，直到读取完整个 HTML 流。
2. **GitHub 精确搜索**:
   - 将查询语句更改为 `q=PackageName+language:R`。
   - 在结果列表中对 `name` 进行不区分大小写的完全匹配验证。
3. **Bioconductor 分类增强**: 增加对 `data/annotation` 等分类的稳定支持。
