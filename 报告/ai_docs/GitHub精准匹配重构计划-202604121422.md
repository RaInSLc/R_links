# GitHub 匹配算法精准进化计划

针对您提出的“优先原名，后驼峰命名”策略，本计划将对 GitHub 检索模块进行深度重构，引入**“评分筛选+反向校验”**的双重过滤模式。

## 方案设计

### 1. 多候选人打分制 (app_window.cpp)
不再直接选取第一个搜索结果。系统将提取结果中的前 5 个候选仓库，并根据仓库名（Repo Name）进行智能评分：
- **第一优先级 (100分)**：仓库名与 R 包原名完全一致（忽略大小写）。解决 `IOBR` 匹配问题。
- **第二优先级 (90分)**：仓库名为包名的驼峰式、连字符或下划线变体。解决 `SeuratDisk` 匹配问题。
- **排除项**：语言非 R 或仓库名带有明显非源码标记（如 `feedstock`、`conda`）的项目将自动降权。

### 2. DESCRIPTION 强行溯源校验 (The Double-Check)
在锁定最高分候选仓库后，系统**必须**执行以下深度校验：
- **反向探测**：抓取该仓库的 `DESCRIPTION` 文件。
- **二次确认**：检查描述文件中的 `Package:` 字段是否确实等于我们搜索的包名。
- **自动降级**：如果校验失败，系统会弃用该仓库并自动尝试评分表中的下一个候选人。

## 验证计划
### 攻坚回测
- **IOBR**：应能避开 ioBroker，由于名称完全匹配且带有 DESCRIPTION，将精准锁定 `IOBR/IOBR`。
- **SeuratDisk**：由于名称形态相似且反向校验成功，将锁定 `mojaveazure/seurat-disk`。
- **estimate**：应能排除非 R 工具，锁定对应的 R 包源码。
