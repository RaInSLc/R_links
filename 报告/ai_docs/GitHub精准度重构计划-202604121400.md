# GitHub 检索精准度重构计划

针对目前体检系统中出现的“为了兼容而兼容”导致的偏差问题（如将 R 包 `IOBR` 误识别为 `ioBroker`），本计划将对检索算法进行底层重构，从“模糊星标优先”转向“精确语义优先”。

## 方案设计

### 1. 强制性检索限制 (app_window.cpp)
- **严格语言过滤**：在 API 请求中强制注入 `language:R` 限制。
- **限制搜索域**：将搜索范围限制在 `in:name,description`，避免匹配到 README 中的随机字符。
- **改进排序**：移除强制星标排序，改用 GitHub 默认的 **Best Match**，确保包名契合度权重高于项目热度权重。

### 2. 智能候选过滤与校验
- **名称精确校验**：在 C++ 层增加对比逻辑。
    - 优先选择仓库名或路径中包含且仅包含包名（忽略大小写和连字符）的结果。
    - 坚决排除像 `iobroker` 这种虽然包含 `iobr` 但语义完全不相关的结果。
- **描述文件二次确认 (Double-Check)**：
    - **逻辑升级**：如果从选定仓库无法解析出合法的 R 包 `DESCRIPTION` 文件，系统将**撤销**该匹配。

### 3. IOBR & SeuratDisk 专项攻坚
- **降级重试逻辑**：若第一轮搜索失败，自动尝试 `PackageName-r` 等变体。
- **黑白名单策略**：在过滤逻辑中增加对知名非 R 项目的启发式避让。

## 验证计划
### 深度对标测试
- **IOBR**：必须锁定 `IOBR/IOBR`，坚决排除 `ioBroker/ioBroker`。
- **estimate**：必须锁定 `KaiAragaki/tidyestimate` 等 R 包。
- **SeuratDisk**：必须锁定 `mojaveazure/seurat-disk`。
