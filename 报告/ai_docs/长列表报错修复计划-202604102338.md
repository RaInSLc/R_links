# 修复 Check System 长列表脚本报错计划

用户在处理超长包列表（约 300+ 个）并使用“全量检查”功能时，生成的 R 脚本在粘贴到 R 控制台时发生报错。这是因为生成的 `packages_to_check` 列表在一行内过长，超出了部分 R 控制台（如 Windows RGui）的单行字符限制（通常为 4096 或 8192），导致语句被截断，从而引发“意外符号”或“括号/引号未闭合”的错误。

## 方案设计

### 1. 优化列表生成逻辑 (pkg_logic.cpp)
修改 `PkgLogic::GenerateBatchCheckList` 函数：
- 每生成一定数量（例如 10 个）的包名后，强制插入一个换行符 `\r\n  `。
- 这样可以将原来几千字符的长行拆分为多个易于解析的短行。

### 2. 改进输出预览格式 (app_window.cpp)
稍微调整 `CheckSystem` 分支的输出模板，增加缩进对齐，使生成的脚本更美观且易于阅读。

## 验证计划
### 自动化验证
- 重新运行 `build.bat` 确保编译无误。
### 手动验证
- 复制用户提供的超长列表。
- 切换到 `Check System` 模式。
- 确认预览框中的 `packages_to_check` 列表是分号折行的。
- 粘贴到 R 环境中验证是否运行成功而不报错。
