# UI字体彻底升级方案总结

> 生成日期: 2026-06-14 | 类型: 总结 | 状态: 已确认

## 背景

用户反馈在此前补充了中文字体回退配置后，UI字体“还是老样子”。这可能由于系统原生字体（如宋体 SimSun 或早期版本的雅黑）在某些显示器上的默认表现依旧不理想，或者此前修改后应用程序未完全重启导致样式没生效。为了彻底改善视觉体验，按照现代网页设计规范（Modern Typography），将应用的字体栈全面替换为更加优雅和清晰的 Google Fonts 方案。

## 问题分析

### 现状
- 原生 Windows 系统字体（Segoe UI / Microsoft YaHei UI）在禁用抗锯齿或作为等宽字体环境下的回退字体时，表现依旧不够一致。
- 尤其是涉及到中文说明（如卡片说明、日志输出）时，原生的 Webview 渲染可能会降级，导致强烈的锯齿感和模糊（发糊）。

### 根因
没有在 Web 层面上强制加载高质量的现代化无衬线 Web 字体，而是完全依赖系统底层已安装字体的渲染机制。

## 核心发现

1. 使用云端托管的现代字体库（如 Inter 和 Noto Sans SC）可以彻底切断 Web 页面对本地系统字体渲染效果的依赖。
2. Noto Sans SC (思源黑体) 在所有字号、所有抗锯齿状态下的表现都远胜于强制后退渲染的系统点阵宋体。

## 实施方案

### 方案概述
引入 Google Fonts 提供的现代字体 `Inter`（针对英文字符和数字）与 `Noto Sans SC`（针对中文字符），彻底重构 `index.html` 的字体加载链和 `App.css` 的全局字体栈。

| 2    | 使用脚本对 `App.css` 的字体栈进行全局替换，将原有的 `Segoe UI`, `Microsoft YaHei UI` 替换为 `'Inter', 'Noto Sans SC'` | 前端样式 (mod_UI) | 低 |
| 3    | 进一步加大界面的基础字号（从 11/12px 继续向 13/14px 调整）以匹配 Noto Sans 的字符高度 | 前端样式 (mod_UI) | 低 |

### 预期效果
- **英文显示**：使用极其清晰、几何感极强的 `Inter` 字体。
- **中文显示**：全部由 `Noto Sans SC`（思源黑体）渲染，笔画均匀、无论是否开启平滑抗锯齿都不会产生断层或模糊。
- **整体阅读体验**：字号更加充实，符合现代应用的设计标准。

## 最终建议

若替换为思源黑体后用户仍觉发糊，则必然是显示器 DPI 物理限制或 Windows 系统层面的缩放设置（DPI Scaling）导致。在代码层面，这已经是最高等级的字体优化方案。
