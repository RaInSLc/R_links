# 修改记录 - UI 交互逻辑优化（体检后自动回归 Auto 模式）

**日期**: 2026-04-10
**时间**: 14:52
**类型**: UI/UX 优化

## 问题背景
用户反馈在“全量体检”完成后，应用程序会自动跳转并选中“Check System (Batch Verify)”模式，而非保持或回到“Auto (Best Effort Smart Routing)”模式。这会导致体检后生成的脚本并非用户期望的智能路由脚本。

## 修改内容
1.  **修复 `app_window.cpp` 中的 `WM_SEARCH_COMPLETE` 逻辑**：
    - 将 `CheckRadioButton` 的目标 ID 从 `IDC_RADIO_CHECK` 更改为 `IDC_RADIO_AUTO`。
    - 将内存中的 `m_currentMethod` 状态从 `InstallMethod::CheckSystem` 更改为 `InstallMethod::Auto`。
2.  **触发命令更新**：在切换回 `Auto` 模式后，立即调用 `UpdateGeneratedCommand` 确保预览框中的脚本根据最新的体检结果和智能路由算法进行更新。

## 验证结果
- 重新编译生成 `RLinkInstaller_PRO.exe`。
- 问题已解决：现在体检结束后，单选框会自动回到“Auto”位置。
