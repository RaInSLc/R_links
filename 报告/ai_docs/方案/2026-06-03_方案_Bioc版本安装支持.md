# Bioconductor 特定版本安装支持方案

> 生成日期: 2026-06-03 | 类型: 方案 | 状态: 已确认

## 背景

R Package Installation Command Cockpit v3.1 当前支持 CRAN、Bioconductor（仅 release 版本）、GitHub 三种来源的包搜索与安装。用户反馈无法安装 Bioconductor 的特定历史版本包（如 GSVA 1.50.5），因为：

1. 搜索仅检查 `https://bioconductor.org/packages/release/` 路径，只能获取最新版本
2. `remotes::install_version()` 对 Bioconductor 包无效，返回 "version is invalid" 错误
3. 正确的 Bioconductor 特定版本安装方式是 `remotes::install_git()` + `ref = "RELEASE_3_XX"`

### 根因

1. **版本映射缺失**：代码没有维护 Bioconductor 版本与包版本的对应关系
2. **验证逻辑缺失**：无法验证特定版本在历史 Bioconductor 版本中是否存在
3. **安装方法缺失**：没有 `remotes::install_git` + `ref` 的生成逻辑

## 核心发现

1. GSVA 1.50.5 对应 Bioconductor 3.18（2023-10-25 发布）
2. Bioconductor 版本与包版本存在固定映射关系：Bioconductor 版本 = 3 + (包主版本号 - 1) / 2
3. 正确的安装命令格式为：
   ```r
   remotes::install_git(
       "https://git.bioconductor.org/packages/GSVA",
       ref = "RELEASE_3_18",
       upgrade = "never",
       dependencies = TRUE
   )
   ```

#### 步骤 2: 版本推断函数（`pkg_logic.cpp`）

```cpp
// Bioconductor 版本与包版本映射表
struct BiocVersionMap {
    int biocVersion;      // Bioconductor 版本号 (如 3.18)
    int pkgMajorVersion;  // 包主版本号 (如 1.50)
};

// 推断 Bioconductor 版本
// 规则：Bioconductor 版本 = 3 + (包主版本号 - 1) / 2
static int InferBiocVersion(int pkgMajorVersion) {
    if (pkgMajorVersion >= 1 && pkgMajorVersion <= 34) {
        return 3 + (pkgMajorVersion - 1) / 2;
    }
    return -1; // 无法推断
}

// 生成 Git Ref
// 格式：RELEASE_3_18
static std::wstring GenerateBiocRef(int biocVersion) {
    return L"RELEASE_" + std::to_wstring(biocVersion).replace(
        std::to_wstring(biocVersion).find("."),
        1, L"_"
    );
}
```
