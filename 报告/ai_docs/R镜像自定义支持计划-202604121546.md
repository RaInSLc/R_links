# 自定义 R 镜像 (Mirror) 支持计划

目前系统生成的安装指令固定使用了官方镜像 `https://cloud.r-project.org`。为了支持 R-Forge 或加速镜像（如清华、阿里镜像），本计划将引入镜像源自定义功能。

## 方案设计

### 1. UI 界面增强 (resource.h & app_window.cpp)
- **设置页扩展**：在 Settings 选项卡中新增“R 镜像地址”输入框。
- **默认值设置**：初始默认为官方镜像 `https://cloud.r-project.org`。
- **资源 ID 分配**：`IDC_ENTRY_MIRROR` (1023)。

### 2. 配置持久化与加载
- **config.ini**：增加 `CranMirror` 字段，确保用户配置在重启后依然有效。

### 3. 指令生成逻辑重构 (pkg_logic.cpp)
- **参数化注入**：修改 `GenerateCommand` 函数，增加镜像参数。
- **动态拼接**：生成的 `remotes::install_version` 和 `install.packages` 的 `repos` 参数将动态引用此配置。

## 验证计划
### 场景回测
- 将设置中的镜像地址修改为 `http://R-Forge.R-project.org`。
- 在 Workspace 中生成 `estimate` 的安装指令。
- **预期输出**：生成的代码中 `repos` 参数应准确反映为 R-Forge 地址。
