# UI字体显示模糊问题修复总结

> 生成日期: 2026-06-14 | 类型: 总结 | 状态: 已确认

## 背景

在 R Package Command Center (mod_UI) 中，用户反馈“大部分字体太虚了 看不清楚”。在前端界面特别是 Windows 环境下，这往往与 CSS 的字体平滑抗锯齿配置以及基础字号设定有关。

## 问题分析

### 现状
当前 `App.css` 的全局样式设置了：
- `-webkit-font-smoothing: antialiased;`
- `text-rendering: optimizeLegibility;`
这导致基于 Chromium 的浏览器内核在渲染深色或浅色文本时，字体的笔画会被显著削薄。在 Windows 的默认字体（如 Segoe UI 或 Microsoft YaHei UI）和低 DPI 屏幕上，表现为字体发虚、对比度低。

### 根因
Mac 开发环境下常用的 `-webkit-font-smoothing: antialiased` 能够让高分屏下的粗体文字显得细腻，但在 Windows ClearType 的亚像素渲染引擎下，该属性会关闭亚像素平滑，从而导致普通字重的文字变得非常细且虚。

## 核心发现

1. `App.css` 的全局 `:root` 和主题容器包含了不适合 Windows 环境的抗锯齿配置。
2. 界面中多次使用了 9px、10px、11px 等极小字号，这进一步放大了平滑算法导致的笔画丢失问题。

## 实施方案

### 方案概述
移除强制抗锯齿 CSS 属性，让浏览器回退到系统默认的字体渲染引擎（即开启 Windows ClearType 亚像素平滑），使得字体渲染回归饱满、清晰。

### 详细步骤

| 步骤 | 操作 | 负责模块 | 风险等级 |
|------|------|----------|----------|
| 1    | 删除 `App.css` 中的 `font-synthesis`、`text-rendering` 和 `-webkit-font-smoothing` 属性 | 前端样式 (mod_UI) | 低 |
| 2    | 提交修改到 Git | 项目代码管理 | 低 |
| 3    | 重新编译 Tauri 客户端产物 | 前端打包与构建 | 低 |
| 4    | 将构建好的 exe 文件复制到 release 目录 | 产物发布 | 低 |

### 预期效果
所有的前端文本，尤其是极小字号或浅色文本，将利用 Windows 自带的 ClearType 进行渲染，笔画变粗，可读性显著提升，消除发虚感。

## 风险评估

| 风险项 | 影响范围 | 可能性 | 缓解措施 |
|--------|----------|--------|----------|
| Mac 下文字略微显粗 | macOS 用户界面 | 中 | 若未来对多端体验有极高要求，可通过 `@supports` 或针对操作系统检测动态加载该 CSS 类 |

## 最终建议

已清理会导致字体发虚的全局 CSS 属性。如果后续界面上部分 9px/10px 的辅助文本依然难以辨认，建议从设计上统一增加全局字体基础字号大小（Base Font Size）。
