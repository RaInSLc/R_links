# 智能路由增强与逻辑修复计划 (解决 CRAN 包识别错误)

针对“survdistr”包（已识别为 CRAN）生成了错误的 `install_url` 指令的问题，我们经过排查确认了原因并制定了如下修复计划。

## 方案设计

### 1. 修复指令回退逻辑
- **移除硬编码重定向**：彻底清理掉那行导致 `Auto` 模式强制回退到 `install_url` 的冗余代码。
- **清理重复行**：修正由于编辑失误导致的指令生成代码重复。

### 2. 精准化智能路由
- **CRAN 路由优化**：
    - 当包源确认为 **CRAN** 时：
        - 如果存在干净的版本号（如 `0.0.3`），强制切换到 `RemotesVersion` 模式（生成 `remotes::install_version`）。
        - 如果没有版本号，回退到 `Base` 模式（生成标准的 `install.packages`），这能确保镜像设置（Repos）生效。
- **保持特殊路由**：确保 GitHub 和 Bioconductor 包依然沿用各自的高级安装函数。

## 验证计划
- **单包回归测试**：输入 `survdistr`，确认生成的指令包含 `install_version`。
- **批量兼容性测试**：输入混合包名（GitHub、Bioc、CRAN），确认每行注释与指令精准匹配。
